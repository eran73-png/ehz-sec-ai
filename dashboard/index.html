🔒 Hardening Level — threat monitoring & detection
Level 0
⚫ OFF
Monitor only — no Telegram or alerts. Useful for debugging.
Level 1
🟢 SOFT
Default. HIGH and CRITICAL alerts only. No INFO or MEDIUM sent.
Level 2
🟡 STRICT
Additional strict rules: pastebin, telnet, FTP, chmod 777, sudo, /dev/tcp.
Level 3
🔴 LOCKDOWN
Alert on every action including INFO. Full investigation mode — maximum verbosity.
📖 What happens at each level
Level Telegram Recommended use
⚫ OFF Nothing Debug / Dev
🟢 SOFT HIGH + CRITICAL Daily use ✓
🟡 STRICT HIGH + CRITICAL Sensitive env
🔴 LOCKDOWN Everything incl. INFO Incident investigation
ℹ️ All levels log to DB — the difference is what is sent via Telegram/Email.
🔒 Monitor only — blocking will be enabled in Milestone 10.

🔔 Notifications — Telegram & Email
✈️
Telegram Bot
Send real-time alerts to a Telegram bot
ℹ️ Telegram settings (Token + Chat ID) are stored in collector/server.js — lines TELEGRAM_TOKEN and TELEGRAM_CHAT_ID.
Here you can only enable/disable sending.
📱 How to create a Telegram bot (phone or desktop):
1. Open Telegram and search for @BotFather
2. Send /newbot
3. Give the bot a name (e.g.: MySecurityBot)
4. BotFather will send you a Token — — copy it to server.js
5. Open your bot and send /start
6. To get your Chat ID — send a message to your bot then open in browser:
   https://api.telegram.org/botTOKEN/getUpdates
📧
Email (SMTP)
Alternative to Telegram — send to any email address
⚠️ To receive email alerts — fill in the details below and enable the toggle above.
Without filling in the details, no alerts will be sent even if the toggle is enabled.
💡 Gmail: Enable "App Passwords" in Google Account, use smtp.gmail.com:587
💡 Outlook: smtp-mail.outlook.com:587 (TLS Off)

What is recorded here?
Every Claude Code conversation is recorded as a Session — a group of actions under the same session ID.
Recorded: Which tools were used (Read/Edit/Bash...), on which files, when, and the risk level.
Not recorded: Prompt text or responses — only the actual actions.
