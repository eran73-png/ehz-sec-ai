מה זה FlowGuard?

הבנת המוצר, מה הוא עושה ולמה צריך אותו

🛡️ בעיה שאנחנו פותרים

Claude Code הוא כלי AI חזק מאוד — הוא יכול לכתוב קוד, להריץ פקודות, לגשת לקבצים ולתקשר עם שירותים חיצוניים. אבל מה קורה ברגע שהוא פועל בשמך?

FlowGuard יושב בין Claude Code לבין המחשב שלך ועוקב אחרי כל פעולה — בזמן אמת.

דוגמה
Claude Code מריץ בשעות הלילה פקודת curl | bash שמורידה סקריפט מהאינטרנט ומריצה אותו. FlowGuard מזהה את זה ושולח לך התראה לטלגרם.

התקנה

מ-Setup.exe ועד שה-Dashboard פתוח ומנטר

📋 דרישות מערכת

  • Windows 10 / 11 (64-bit)
  • Claude Code CLI מותקן (claude בטרמינל)
  • הרשאות Administrator להתקנה
  • ~50MB דיסק פנוי
טיפ
Node.js מגיע bundled בתוך ה-installer — אין צורך להתקין אותו בנפרד.

🚀 שלבי ההתקנה

הרץ את FlowGuard-Setup-v2.9.3.exe

לחץ ימני → "הפעל כמנהל". אשר את חלון ה-UAC.

בחר תיקיית התקנה

ברירת מחדל: C:\FlowGuard\. ניתן לשנות.

בחר תיקיית פרויקט (Project Directory)

ה-installer מבקש לבחור את תיקיית הפרויקט שלך. זוהי התיקייה ש-FlowGuard מנטר. לחץ Browse ובחר את תיקיית הקוד שלך (למשל C:\Claude-Project). שדה זה הוא חובה.

סיום — FlowGuard פועל

ההתקנה מסתיימת אוטומטית — Hooks מותקנים, ה-Collector מופעל כ-Windows Service, ו-Dashboard נפתח בדפדפן.

טיפ
הגדרות Telegram ו-Hardening Level ניתנות להגדרה מאוחר יותר דרך Settings ב-Dashboard.

✅ בדיקה שהכל עובד

  • ב-System Tray — אייקון FlowGuard עם עיגול ירוק 🟢
  • בפתיחת Dashboard — כתוב "Collector Active" בצבע ירוק
  • הרץ פקודה כלשהי ב-Claude Code ובדוק שמגיע event ב-Dashboard

📊 מסך Events — איך לקרוא

כל שורה בטבלה מייצגת פעולה אחת שקלוד ביצע:

עמודהמה זה
Timeתאריך ושעה מדויקים של הפעולה
Levelרמת החומרה: CRITICAL / HIGH / MEDIUM / INFO
Toolהכלי שבו קלוד השתמש (Edit, Bash, WebFetch...)
Descriptionתיאור מה בוצע. אם ארוך — לחץ "show more ▼" לפרוט מלא

טיפ — סינון Events
השתמש בתפריטי הסינון למעלה: בחר רמה (CRITICAL בלבד), כלי (FSWatcher), או הקלד חיפוש חופשי. הדרופדאון "Last 500 / All" שולט בכמה events נטענים.

🔗 Audit Chain — שלמות הרשומות

FlowGuard שומר hash של כל event ומחבר אותו ל-hash הקודם (כמו blockchain). כך אפשר לאמת שאף event לא נמחק או שונה.

  • Chain Valid ✅ — כל הרשומות תקינות ולא שופרו
  • Chain Broken ❌ — מישהו מחק או שינה event ביד
שים לב
לחיצה על "Reset Chain" מאפסת את השרשרת ומתחיל מחדש — פעולה זו בלתי הפיכה.

File Audit

סריקת קבצים לגילוי סקרטים, מפתחות API ומידע רגיש

🔍 מה File Audit עושה?

סורק קבצי קוד בתיקיית הפרויקט ומחפש מידע רגיש שלא אמור להיות בקוד — API keys, סיסמאות, מספרי כרטיס אשראי, ועוד.

הסריקה כוללת גם קבצי Office (.docx, .xlsx, .pptx), קבצים ללא סיומת (כמו VISA, passwords), ותיבת חיפוש לסינון לפי שם קובץ (כולל עברית).

למה זה חשוב?
קוד שמועלה ל-GitHub עם API key בפנים — גנוב תוך דקות. FlowGuard מזהה את זה לפני שזה קורה.

🛠️ כיצד לבצע סריקה

בחר תיקייה

לחץ "Browse" לבחור תיקייה, או השאר את ברירת המחדל (תיקיית הפרויקט).

בחר סוג סריקה

Full Scan — כל הקבצים. Changed Only — רק קבצים שהשתנו מאז הסריקה האחרונה (מהיר יותר).

לחץ "Scan Now"

הסריקה רצה ברקע. התוצאות מופיעות בטבלה.

📖 קריאת תוצאות

כל שורה בטבלה = קובץ עם ממצא. לחץ על השורה להרחיב ולראות פרטים:

  • Reason — סוג הממצא (למשל: "Hardcoded password")
  • Explain — הסבר בשפה פשוטה מה הסכנה
  • Line — מספר שורה בקובץ
  • Snippet — חלק מהקוד שבו נמצאה הבעיה
ממצאהמשמעותמה לעשות
CRITICAL OpenAI API keyמפתח API חשוף בקודבטל את המפתח מיידית, עבור למשתני סביבה (.env)
CRITICAL curl|shהורדה והרצה של סקריפט מהאינטרנטבדוק מה הסקריפט עושה ואם הכנסת אותו בכוונה
HIGH Hardcoded passwordסיסמה כתובה ישירות בקודהעבר למשתני סביבה או קובץ .env מחוץ ל-git
HIGH JWT tokenטוקן אימות hardcodedהסר מהקוד — יוצר בזמן ריצה, לא נשמר
MEDIUM eval()הרצת קוד דינמיבדוק אם הקלט מהימן — החלף ב-JSON.parse אם אפשר

⚙️ אפשרויות נוספות

  • Hide build/generated files — מסנן קבצים של node_modules, dist וכו' שמייצרים הרבה false positives
  • Export Excel — ייצוא כל הממצאים לקובץ CSV לדיווח

Git Monitor

עקיבה אחרי פעולות Git ומניעת דליפת קוד

🌐 Git Remote Monitor

רשימת השרתים שמורשים לקבל git push מהמחשב שלך. אם Claude Code ינסה לדחוף קוד לשרת שלא ברשימה — תקבל התראה.

שרתמה זה
github.comGitHub — שירות Git הפופולרי ביותר
gitlab.comGitLab — אלטרנטיבה ל-GitHub
bitbucket.orgBitbucket — שירות Git של Atlassian
localhost / 127.0.0.1שרת Git מקומי על המחשב שלך
שרת מותאם אישית?
אם יש לך שרת Git פרטי (למשל ehz-server.duckdns.org) — הוסף אותו ב-Settings → Authorized Git Remotes.

📜 Git Events

היסטוריה של כל פעולות ה-push/fetch שקלוד ביצע — מתי, לאיזה שרת, ואיזו פקודה הורצה.

מדוע ריק?
Git Events מתמלא רק כשקלוד מריץ פקודות git דרך ה-Hook. אם עשית push ישירות מהטרמינל — זה לא יתועד כאן.

⚠️ מה קורה כשמזוהה push לא מורשה?

  • נוצר event ברמת HIGH בטבלת Events
  • נשלחת התראה לטלגרם (אם מוגדר)
  • ב-HARD mode — הפקודה עלולה להיחסם

Settings — כל כפתור

הסבר מלא על כל הגדרה ומה היא עושה

📂 Project Directory

תיקיית השורש ש-FlowGuard מנטר. מוגדרת בזמן ההתקנה, וניתנת לשינוי ב-Settings.

לחיצה על כפתור Browse פותחת בוחר תיקיות בצד השרת ומאפשרת לבחור תיקייה חדשה. כל הניטור — FSWatcher, File Audit, Projects Explorer — מבוסס על תיקייה זו.

טיפ
שינוי Project Directory דורש Restart ל-Collector כדי שה-FSWatcher יתחיל לנטר את התיקייה החדשה.

🌙 Quiet Hours

בשעות שקטות (ברירת מחדל: 23:00–07:00) — FlowGuard ממשיך לתעד events ל-DB, אבל לא שולח הודעות לטלגרם.

שימושי
גיבויים לילה מריצים הרבה פעולות על קבצים — ללא Quiet Hours תקבל עשרות הודעות בלילה. עם Quiet Hours הכל נשמר ואתה רואה בבוקר.
הגדרההסבר
Enable Quiet Hoursהפעל/כבה את כל הפיצ'ר
Fromשעת התחלה של הדממה (ברירת מחדל: 23:00)
Toשעת סיום הדממה (ברירת מחדל: 07:00)

📂 FSWatcher Exclusions

תיקיות שה-FSWatcher מתעלם מהן לחלוטין. שימושי כדי למנוע false positives מתיקיות שמשתנות כל הזמן.

ב-Settings ישנם שני כרטיסים: Quiet Hours ו-FSWatcher Exclusions. כל תיקייה מומלצת מוצגת עם toggle — המשתמש מחליט מה לכלול ומה לא.

  • מומלצות — ON כברירת מחדל: node_modules, .git, dist
  • מומלצות — OFF (המשתמש מחליט): build, out, .next, coverage, .cache, tmp
  • Custom: הוסף כל תיקייה אישית דרך שדה ה-Add בתחתית הכרטיס
  • כל השינויים נשמרים אוטומטית ב-whitelist.json ונכנסים לתוקף מיידית

🧠 Behavioral Learning

FlowGuard לומד דפוסי עבודה ומפחית false positives אוטומטית:

  • קובץ שנגעה בו Claude בסשן הזה → INFO בלבד
  • קובץ שמשתנה 3+ פעמים ביום → INFO
  • קובץ שמשתנה 10+ פעמים → לא מוצג (suppress)
  • גיבוי לילי, scheduled task — מזוהה אוטומטית → INFO

רק קבצים חדשים לגמרי שלא נראו קודם מקבלים LOW/HIGH.

🔗 Authorized Git Remotes

רשימת שרתי Git שמורשים לקבל push. כל שרת שלא ברשימה ייצור התראה.

להוספת שרת פרטי: הקלד חלק מהכתובת (למשל ehz-server.duckdns.org) ולחץ Add.

Projects Explorer ו-Web Access

תצוגת מבנה הפרויקט וניטור גישה לאינטרנט

📁 Projects Explorer

מציג את כל תת-התיקיות בתיקיית הפרויקט (Project Directory) עם מספר הקבצים בכל תיקייה. מאפשר לראות במבט מהיר את מבנה הפרויקט ולזהות תיקיות גדולות או חריגות.

  • כל תיקייה מוצגת עם שם, מספר קבצים, וגודל כולל

FAQ

FlowGuard מאט את Claude Code?

לא. ה-Hook מוסיף פחות מ-5ms לכל פעולה. אין השפעה מורגשת על הביצועים.

מה קורה אם ה-Collector מפסיק לעבוד?

Claude Code ממשיך לעבוד בלי בעיה — ה-Hook יכשל בשקט אם אין Collector. לא יהיה ניטור עד שה-Collector יחזור לפעול. ה-Windows Service מוגדר לאתחל אוטומטית אחרי כשל.

האם FlowGuard שולח מידע לאינטרנט?

לא. כל הנתונים נשמרים מקומית על המחשב שלך. התקשורת היחידה החוצה היא התראות Telegram (שאתה מפעיל ומגדיר) ועדכון חוקים מ-GitHub (ניתן לכיבוי).

קיבלתי CRITICAL על גיבוי לילי — מה עושים?

הגדר Quiet Hours (Settings → Quiet Hours) לשעות הגיבוי. הגיבויים ימשיכו להירשם ב-DB אבל לא ישלחו התראות טלגרם.

File Audit מצא API key — מה עושים?

  1. בטל את המפתח מיידית בממשק של הספק (OpenAI, AWS וכו')
  2. צור מפתח חדש
  3. העבר את המפתח לקובץ .env וודא שהוא ב-.gitignore
  4. בקוד — השתמש ב-process.env.API_KEY במקום הערך הישיר

File Audit מראה "No audit data" — מה עושים?

לחץ על כפתור Scan Now כדי להפעיל סריקה. ודא שהנתיב ב-File Audit תואם את ה-Project Directory שהגדרת — אם הם לא תואמים, הסריקה לא תמצא קבצים.

היסטוריית Web Access ריקה — למה?

היסטוריית WebFetch מוצגת רק כש-Claude Code מבצע בקשות web דרך ה-Hooks. גלישה רגילה בדפדפן לא נרשמת — רק פעולות WebFetch ו-WebSearch ש-Claude Code מבצע מתועדות כאן.
