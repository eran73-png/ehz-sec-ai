What is FlowGuard?

Understanding the product, what it does, and why you need it

🛡️ The Problem We Solve

Claude Code is an extremely powerful AI tool — it can write code, run commands, access files, and communicate with external services. But what happens the moment it acts on your behalf?

FlowGuard sits between Claude Code and your machine, tracking every action in real time.

Example
Claude Code runs a curl | bash command late at night, downloading a script from the internet and executing it. FlowGuard detects this and sends you an alert on Telegram.

⚙️ How FlowGuard Works

  • Hook — Registered in Claude Code settings. Every action Claude performs passes through it.
  • Collector — A Node.js server running in the background as a Windows Service. Receives events, saves them to the DB, and sends alerts.
  • Dashboard — A visual interface for viewing all activity, managing settings, and performing security analysis.

Installation

From Setup.exe to an open, monitoring Dashboard

📋 System Requirements

  • Windows 10 / 11 (64-bit)
  • Claude Code CLI installed (claude in terminal)
  • Administrator privileges for installation
  • ~50MB free disk space
Tip
Node.js is bundled inside the installer — no need to install it separately.

🚀 Installation Steps

Run FlowGuard-Setup.exe

Right-click → "Run as administrator". Approve the UAC prompt.

Choose installation folder

Default: C:\FlowGuard\. You may change this.

Config Wizard — Configure Telegram

Enter your Bot Token and Chat ID to receive alerts. You can skip this and configure it later via the Dashboard.

Choose Hardening Level

SOFT (monitoring only) / MEDIUM / HARD (active blocking). Start with SOFT.

Done — FlowGuard is running

The Collector starts as a Windows Service. The FlowGuard icon appears in the System Tray. Open the Dashboard from the tray icon.

✅ Verify Everything Works

  • In the System Tray — FlowGuard icon with a green circle 🟢
  • When opening the Dashboard — "Collector Active" is shown in green
  • Run any command in Claude Code and verify an event appears in the Dashboard

📊 Reading the Events Screen

Each row in the table represents one action that Claude performed:

ColumnWhat It Is
TimeExact date and time of the action
LevelSeverity level: CRITICAL / HIGH / MEDIUM / INFO
ToolThe tool Claude used (Edit, Bash, WebFetch…)
DescriptionDescription of what was done. If long — click "show more ▼" for full details

Tip — Filtering Events
Use the filter dropdowns at the top: choose a level (CRITICAL only), a tool (FSWatcher), or type a free-text search. The "Last 500 / All" dropdown controls how many events are loaded.

🔗 Audit Chain — Record Integrity

FlowGuard stores a hash for every event and links it to the previous hash (similar to a blockchain). This allows you to verify that no event has been deleted or altered.

  • Chain Valid ✅ — All records are intact and unmodified
  • Chain Broken ❌ — Someone deleted or manually modified an event
Note
Clicking "Reset Chain" resets the chain and starts fresh — this action is irreversible.

File Audit

Scanning files to detect secrets, API keys, and sensitive information

🔍 What Does File Audit Do?

Scans code files in a folder you choose, looking for sensitive information that should not be in code — API keys, passwords, credit card numbers, and more.

Why This Matters
Code pushed to GitHub with an API key inside can be stolen within minutes. FlowGuard catches this before it happens.
✨ New in v2.1.3 — Extended Scanning
  • Office files — scans .docx, .xlsx, .pptx (extracts text from internal ZIP structure)
  • Extensionless files — files like VISA, passwords are now scanned too
  • Search box — filter results by filename, including Hebrew filenames

🛠️ How to Run a Scan

Select a folder

Click "Browse" to choose a folder, or leave the default (C:/Claude-Repo).

Choose scan type

Full Scan — all files. Changed Only — only files that changed since the last scan (faster).

Click "Scan Now"

The scan runs in the background. Results appear in the table.

📖 Reading Results

Each row in the table = a file with a finding. Click the row to expand and see details:

  • Reason — Type of finding (e.g., "Hardcoded password")
  • Explain — Plain-language explanation of the risk
  • Line — Line number in the file
  • Snippet — The portion of code where the issue was found
FindingMeaningWhat to Do
CRITICAL OpenAI API keyAPI key exposed in codeRevoke the key immediately, switch to environment variables (.env)
CRITICAL curl|shDownloading and executing a script from the internetVerify what the script does and whether you intentionally added it
HIGH Hardcoded passwordPassword written directly in the codeMove to environment variables or a .env file outside of git
HIGH JWT tokenHardcoded authentication tokenRemove from code — generate at runtime, do not store it
MEDIUM eval()Dynamic code executionCheck whether the input is trusted — replace with JSON.parse where possible

⚙️ Additional Options

  • Hide build/generated files — Filters out node_modules, dist, and similar directories that generate many false positives
  • Export Excel — Export all findings to a CSV file for reporting

Git Monitor

Tracking Git operations and preventing code leaks

🌐 Git Remote Monitor

A list of servers authorized to receive git push from your machine. If Claude Code attempts to push code to a server not on this list — you will receive an alert.

ServerWhat It Is
github.comGitHub — the most popular Git service
gitlab.comGitLab — an alternative to GitHub
bitbucket.orgBitbucket — Atlassian's Git service
localhost / 127.0.0.1A local Git server on your own machine
Custom Server?
If you have a private Git server (e.g., ehz-server.duckdns.org) — add it under Settings → Authorized Git Remotes.

📜 Git Events

A history of all push/fetch operations that Claude performed — when, to which server, and which command was run.

Why Is It Empty?
Git Events are only populated when Claude runs git commands through the Hook. If you pushed directly from a terminal — that will not be logged here.

⚠️ What Happens When an Unauthorized Push Is Detected?

  • An event is created at HIGH level in the Events table
  • An alert is sent to Telegram (if configured)
  • In HARD mode — the command may be blocked

Settings — Every Control Explained

A full explanation of every setting and what it does

🔒 Security Settings

SettingDescriptionRecommendation
Hardening Mode SOFT — monitoring only, no blocking.
MEDIUM — blocks suspicious actions with confirmation.
HARD — automatically blocks CRITICAL actions.		 Start with SOFT, move to MEDIUM after one week
Block Critical Actions When enabled — FlowGuard blocks CRITICAL commands (such as curl|bash) before they are executed. Enable in MEDIUM/HARD mode
File System Watcher Monitors changes in the project folder. Detects files that were copied, deleted, or modified without Claude's involvement. Recommended to always keep enabled
Audit Chain Every event receives a hash linked to the previous one. Guarantees integrity — no event can be deleted without detection. Always enabled

📦 Data Settings

SettingDescriptionRecommendation
Log Retention How many days to keep events in the DB. After this period they are deleted automatically. Default: 30 days. 30 days for regular use, 90 for compliance needs
Auto-update Rules FlowGuard automatically updates its detection rules from GitHub. When off — the current rules remain in place. Recommended to always keep enabled

🔔 Alert Settings

SettingDescription
Desktop Alerts Pop-up notification from the System Tray for every anomalous event. Does not require an internet connection.
Telegram Alerts Sends a message to Telegram with event details. Arrives within seconds. Requires a Bot Token + Chat ID.
Min Alert Level The minimum level for sending an alert:
CRITICAL only — only the most severe issues.
HIGH+ — CRITICAL + HIGH.
MEDIUM+ — includes minor anomalies (more noise).

🌙 Quiet Hours

During quiet hours (default: 23:00–07:00) — FlowGuard continues to log events to the DB, but does not send Telegram messages.

Useful
Nightly backups trigger many file operations — without Quiet Hours you will receive dozens of messages overnight. With Quiet Hours everything is saved and you review it in the morning.
SettingDescription
Enable Quiet HoursEnable/disable the entire feature
FromStart time of the silence window (default: 23:00)
ToEnd time of the silence window (default: 07:00)

📂 FSWatcher Exclusions

Folders that FSWatcher ignores completely. Prevents false positives from directories that change constantly.

✨ New in v2.1.4 — Toggle Interface
Settings now has two equal cards: Quiet Hours and FSWatcher Exclusions.
Each recommended folder has a toggle — you decide what to exclude and what to watch.
  • Recommended — ON by default: node_modules, .git, dist
  • Recommended — OFF (your choice): build, out, .next, coverage, .cache, tmp
  • Custom: add any personal folder using the Add field at the bottom of the card
  • All changes save automatically to whitelist.json and take effect immediately
🧠 New in v2.1.4 — Behavioral Learning
FlowGuard learns your work patterns and automatically reduces false positives:
  • File touched by Claude this session → INFO only
  • File changed 3+ times today → INFO
  • File changed 10+ times → suppressed entirely
  • Nightly backup, scheduled task — auto-detected → INFO
Only truly new, unseen files receive LOW/HIGH alerts.

🔗 Authorized Git Remotes

List of Git servers authorized to receive pushes. Any server not on this list will trigger an alert.

To add a private server: type part of the address (e.g., ehz-server.duckdns.org) and click Add.

Setting Up Telegram

Receive alerts directly on your phone

🤖 Creating a Bot

Open @BotFather on Telegram

Search for "BotFather" on Telegram and start a conversation.

Send /newbot

Give the bot a name (e.g., "FlowGuard Bot") and then a username (must end in bot, e.g., "flowguard_mybot").

Receive your Bot Token

BotFather will send a string in the format 123456789:AAFxxxxx... — this is your Token.

Find your Chat ID

Start a conversation with @userinfobot on Telegram → send any message → it will return your ID (a number, e.g., 123456789).

Enter in FlowGuard Settings

Settings → Telegram Token + Chat ID → Save. Click "Test" to verify.

Frequently Asked Questions

FAQ

Does FlowGuard slow down Claude Code?

No. The Hook adds less than 5ms per action. There is no noticeable impact on performance.

What happens if the Collector stops working?

Claude Code continues to work without issue — the Hook will fail silently if there is no Collector. There will be no monitoring until the Collector is back online. The Windows Service is configured to restart automatically after a failure.

Does FlowGuard send any data to the internet?

No. All data is stored locally on your machine. The only outbound communication is Telegram alerts (which you enable and configure yourself) and rule updates from GitHub (which can be disabled).

I received a CRITICAL alert from a nightly backup — what do I do?

Configure Quiet Hours (Settings → Quiet Hours) to cover the backup window. Backups will continue to be logged in the DB but no Telegram alerts will be sent.

File Audit found an API key — what do I do?

  1. Revoke the key immediately in the provider's dashboard (OpenAI, AWS, etc.)
  2. Generate a new key
  3. Move the key to a .env file and make sure it is in .gitignore
  4. In the code — use process.env.API_KEY instead of the literal value
