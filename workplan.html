7.5 ✅
Immutable Audit Log ✅
What it does: Every event saved to the database gets a digital signature (SHA-256 hash). Each signature depends on the previous one — forming a chain.

Why it matters: If someone edits or deletes an event from the database to hide their tracks — the chain breaks. You can instantly prove the log was tampered with. This is legal evidence.

Think of it like this: Each event is a link. You cannot change one link without breaking all the links after it.

In the dashboard: A 🔗 badge in the header shows how many signed events exist. Click it → runs full verification → shows ✅ Chain Valid or ❌ Chain BROKEN with the exact event where tampering occurred.

Tested: Sent 2 events → verified chain intact → confirmed SHA-256 hash chain working. Done: 09.04.2026

הקשחה — Hardening Levels
מה זה הקשחה?
Claude מקבל הגדרת תיקיית פרויקט — וכל גישה מחוצה לה נחסמת, מדווחת, או מצריכה אישור. כמו Firewall Zone — אתה מגדיר מה "פנים" ומה "חוץ".
🔓
Level 0 — OFF
ניטור בלבד, ללא הגבלה
✅ קריאה — כל מקום
✅ כתיבה — כל מקום
✅ WebFetch — כל URL
✅ Bash — כל פקודה
ℹ️ הכל מתועד בלבד
⚠️
Level 1 — SOFT
התראה על חריגה מהפרויקט
✅ קריאה — כל מקום
⚠️ כתיבה מחוץ לפרויקט → alert
⚠️ WebFetch URL חדש → alert
✅ Bash — כל פקודה
📱 Telegram על כל חריגה
🔒
Level 2 — STRICT
חסימה + אישור ידני
✅ קריאה — בתיקיית פרויקט בלבד
❌ כתיבה מחוץ לפרויקט → חסום
⚠️ WebFetch — Whitelist בלבד
❌ Bash חשוד → חסום
🔐 חריגה → אישור ידני נדרש
🔐
Level 3 — LOCKDOWN
נעילה מלאה — פרויקט בלבד
✅ קריאה — בתיקיית פרויקט בלבד
✅ כתיבה — בתיקיית פרויקט בלבד
❌ WebFetch — חסום לחלוטין
❌ Bash — חסום לחלוטין
