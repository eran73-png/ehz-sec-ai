תכנית vs. מחקר — פריט אחר פריט

ירוק = מכוסה מלא · צהוב = חלקי · אדום = חסר · סגול = שיפור חדש
● מכוסה מלא — תיכננו ועונה על המחקר
Hook Architecture — HTTP Collector
PreToolUse/PostToolUse שולחים events ל-Collector מקומי. זה בדיוק ה-Reference Implementation הרשמי של Anthropic. ארכיטקטורה מושלמת.
בתכנית Milestone 1 — hook.js + Collector port 3010
מכוסה מלא ✓
Anomaly Rules Engine
Signature + Context + Behavioral + Whitelist. מכסה curl|sh, rm -rf, base64, ngrok, פעילות חריגה. המחקר אישר שזה הכיוון הנכון.
בתכנית Milestone 2 — rules.js, 4 סוגי חוקים
מכוסה מלא ✓
Secrets + DLP Scanner
סיסמאות, API keys (AWS/GitHub/OpenAI), JWT, כרטיסי אשראי, ת"ז ישראלית. Real-time Write Guard. OWASP LLM06 מכוסה.
בתכנית Milestone 6 — secrets-patterns.json + Write Guard
מכוסה מלא ✓
Skill Scanner — Supply Chain
Hash integrity + YARA + Semgrep + ClamAV + VirusTotal לפני כל התקנת Skill. מכסה OWASP LLM05 (Supply Chain Vulnerabilities).
בתכנית Milestone 5 — 4 מנועים + Quarantine
מכוסה מלא ✓
Hardening Levels 0–3
OFF / SOFT / STRICT / LOCKDOWN. מכסה OWASP LLM08 (Excessive Agency). Telegram /approve + /deny לאישור ידני בLevel 2.
בתכנית Milestone 4 — Project Scope + 4 Levels
מכוסה מלא ✓
Block Mode — עצירת Claude
process.exit(1) עוצר את Claude לחלוטין. מכסה Excessive Agency ברמה הגבוהה ביותר. Exceptions management.
בתכנית Milestone 10 — שלב ג'
מכוסה מלא ✓
Behavioral Monitoring — DoS Prevention
>20 WebFetch/דקה, >50 Read/5 דקות. מכסה OWASP LLM04 (Model Denial of Service) — זיהוי שימוש חריג במשאבים.
בתכנית Milestone 2.4 — Behavioral Rules
מכוסה מלא ✓
Immutable Audit Log
Hash chain לוג — כל entry חתום, לא ניתן לשנות רטרואקטיבית. Audit trail מלא לComplianceוForensics.
בתכנית Milestone 7.5
מכוסה מלא ✓
SIEM Integration
Syslog RFC 5424, Microsoft Sentinel (KQL), Splunk (SPL). ניתוח גלובלי ב-Splunk/Sentinel לאחר המחקר.
בתכנית Milestone 11 — שלב ד'
מכוסה מלא ✓
● כיסוי חלקי — תיכננו, אבל המחקר גילה שצריך יותר
Prompt Injection Detection
תיכננו Context Rules שמזהים patterns חשודים, אבל המחקר (OWASP LLM01) מגדיר Prompt Injection כסיכון נפרד שדורש זיהוי ספציפי בתוכן שClaude מקבל — לא רק בפקודות.
בתכנית Milestone 2.2 — Context Rules (חלקי)
חלקי ⚠️
Insecure Output Handling
ה-Write Guard שלנו סורק secrets, אבל OWASP LLM02 מגדיר גם בדיקה שהקוד שנכתב לא מכניס XSS/SQLi/RCE. Semgrep ב-PostToolUse יכסה את זה — אך לא תיכננו את זה במפורש.
בתכנית Milestone 6.4 — Write Guard (חלקי)
חלקי ⚠️
MCP Server Monitoring
תיכננו Hook כללי לכל tool, אבל Anthropic מדגישים MCP כסיכון נפרד. המחקר הראה שצריך matcher ספציפי לכל MCP write operations — mcp__*__write*.
בתכנית Hook כללי — לא ספציפי ל-MCP
חלקי ⚠️
● חסר לחלוטין — לא תיכננו, עלה מהמחקר
WebDAV Windows Block
Anthropic מזהירים במפורש: paths שמתחילים ב-\\* ב-Windows יכולים לעקוף את Permission System דרך WebDAV. לא תיכננו חוק ספציפי לזה.
בתכנית לא קיים כלל
חסר ❌
ConfigChange Hook
Anthropic מספקים ConfigChange hook — מופעל כשמישהו משנה את settings.json. לא ידענו על זה. חשוב לזהות ניסיון לכבות hooks או לשנות הגדרות אבטחה.
בתכנית לא קיים כלל
חסר ❌
OpenTelemetry Export
Anthropic מספקים OTel מובנה. לא תיכננו לשלב OTel — כך EHZ-SEC-AI לא מדבר עם Prometheus/Grafana/Datadog קיימים בארגון. זה פער לסביבת Enterprise (שלב ב').
בתכנית לא קיים (רלוונטי לשלב ב')
חסר ❌
Training Data / Model Theft Protection
OWASP LLM03 ו-LLM10 — Data Poisoning ו-Model Theft. אלה סיכונים שהמחקר הזכיר. לא ניתן להתגונן מהצד שלנו — זה ברמת Anthropic בלבד. חשוב לתעד את הגבול.
בתכנית לא קיים — מחוץ לסקופ

יתרונות וחסרונות — תמונה מלאה

מה EHZ-SEC-AI עושה טוב, ומה הגבולות שלו
✅ יתרונות ברורים
🎯 ייחודי בשוק — אין מוצר שמנטר Claude Code ספציפית דרך Hooks עם Dashboard + Telegram. Lasso ו-Spectral הם כלליים ויקרים.
✅ Anthropic מאשרים — הארכיטקטורה שלנו (HTTP Hook → Collector) היא Reference Implementation רשמי שלהם. בונים על היסודות הנכונים.
🆓 חינמי לחלוטין — YARA, ClamAV, Semgrep, Google Safe Browsing, VirusTotal (free tier). Lasso עולה עשרות אלפי דולר לשנה.
🇮🇱 PII ישראלי ייחודי — סריקת ת"ז, כרטיסי אשראי, מספרי טלפון ישראליים. אין מוצר אחר שעושה את זה.
📱 Telegram + System Tray — שילוב שלא קיים בשום מוצר. התראה לנייד + אייקון + Dashboard = כיסוי מלא.
🔒 100% על Excessive Agency — Hardening Levels + Block Mode = הכי חזק בשוק לשליטה על אוטונומיית Claude.
🏠 מקומי לחלוטין — הנתונים לא יוצאים לענן. כל הניתוח מקומי. פרטיות מלאה.
⚠️ חסרונות ומגבלות
🔴 WebDAV לא מכוסה — Anthropic מזהירים על סיכון ספציפי ב-Windows שלא תיכננו. דורש תיקון לפני שלב א'.
🟡 Prompt Injection — חלקי — זיהוי בפקודות בלבד. לא בתוכן שClaude מקבל מ-WebFetch/קבצים. OWASP LLM01 הכי נפוץ.
🟡 ConfigChange Hook חסר — לא ידענו על ה-Hook הזה. מישהו יכול לשנות את settings.json ולכבות את ה-Hooks שלנו — בלי שנדע.
🟡 MCP Monitoring כללי מדי — Hook כללי לכל tool. לא matcher ספציפי ל-MCP write operations. עלול להחמיץ פעולות MCP מסוכנות.
⚪ אין OTel Integration — ארגונים עם Prometheus/Grafana לא יכולים לשלב EHZ-SEC-AI בתשתית הקיימת. פגיעה באימוץ ב-Enterprise.
⚪ OWASP LLM03 + LLM10 = 0% — Data Poisoning ו-Model Theft לא ניתן לטפל בהם מצד הלקוח. מגבלה מובנית שצריך לתעד בבירור.
⚪ Node.js תלות — מחייב Node.js על כל מחשב. MSI Installer מטפל בזה, אבל מוסיף מורכבות לפריסה.

המלצות שיפור — מה כדאי להוסיף

5 שיפורים ממוקדים שיעלו את הכיסוי מ-82% ל-95%+
🔴 עדיפות קריטית
הוסף חוק WebDAV לMilestone 2
Anthropic מזהירים: paths שמתחילים ב-\\ ב-Windows יכולים לעקוף Permission System. צריך להוסיף ל-Signature Rules:

if (path.startsWith('\\\\')) → CRITICAL: WebDAV bypass attempt

זה שיפור פשוט של שורה אחת — לא דורש Milestone נפרד.
+ Milestone 2.1 — Signature Rules
🔴 עדיפות קריטית
הוסף ConfigChange Hook
כרגע אפשר לכבות את EHZ-SEC-AI על ידי עריכת settings.json ללא שנדע. ה-ConfigChange hook של Anthropic מתריע על זה:

"ConfigChange" → hook → Telegram alert "⚠️ settings.json שונה!"

בלי זה — האבטחה שלנו ניתנת לעקיפה.
+ Milestone 1.5 — ConfigChange Watcher
🟠 עדיפות גבוהה
שפר Prompt Injection Detection
כרגע Context Rules בודקים רק פקודות. OWASP LLM01 דורש גם בדיקת תוכן שClaude מקבל — מ-WebFetch, קריאת קבצים.

הוסף PostToolUse scanner שבודק אם תוצאת WebFetch/Read מכילה: "ignore previous instructions", "you are now", "jailbreak"
+ Milestone 2.5 — Prompt Injection Scanner
🟠 עדיפות גבוהה
MCP-Specific Hook Matchers
Anthropic מדגישים MCP כווקטור סיכון נפרד. הוסף matchers ספציפיים:

mcp__*__write* → HIGH alert
mcp__*__delete* → CRITICAL
mcp__*__exec* → CRITICAL

כך ניתן לנטר MCP servers בנפרד מ-tools רגילים.
+ Milestone 2 או 5 — MCP Rules
🟡 שיפור מועיל
OpenTelemetry Output — לשלב ב'
ארגונים עם Prometheus/Grafana/Datadog לא יכולים לשלב EHZ-SEC-AI ללא OTel.

הוסף ל-Collector export של events בפורמט OTel סטנדרטי. כך EHZ-SEC-AI "מדבר" עם תשתית ניטור קיימת ולא מחליף אותה.
+ Milestone 9 — Domain/Enterprise

🏆 המסקנה הסופית

התכנית שלנו טובה מאוד — 82% כיסוי ממחקר עצמאי, ארכיטקטורה שAnthropic עצמם מאשרים, ויכולות שלא קיימות בשום מוצר מסחרי. עם 2 תיקונים קריטיים (WebDAV + ConfigChange) ו-2 שיפורים גבוהים (Prompt Injection + MCP) נגיע ל-95%+ כיסוי — ונהיה עדיפים על Lasso ו-Spectral לסביבת Claude Code.
