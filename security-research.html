חלק א׳ — מה אומר Anthropic עצמם על אבטחה
🏛 עמדת Anthropic — עקרונות יסוד
Anthropic מפרסמים תיעוד מלא באתר code.claude.com/docs/en/security. הגישה שלהם: Permission-based Architecture — Claude Code מבצע קריאה בלבד כברירת מחדל. כל פעולה נוספת (כתיבה, פקודות) דורשת אישור מפורש.
🔒 הגנות מובנות
  • ✅ Sandboxed Bash — בידוד filesystem+network
  • ✅ Write restriction — רק בתיקיית הפרויקט
  • ✅ Command blocklist — curl/wget חסומים כברירת מחדל
  • ✅ Fail-closed — פקודה לא מוכרת → דורשת אישור
  • ✅ Isolated WebFetch context — מנגנון נגד Prompt Injection
  • ✅ Command injection detection — פקודות חשודות → אישור ידני
⚠️ מה Anthropic לא עושים
  • ❌ אין Dashboard לניטור אירועים
  • ❌ אין התראות Telegram/מייל על פעילות
  • ❌ אין סריקת סיסמאות/secrets אוטומטית
  • ❌ אין Hardening Levels מוגדרים
  • ❌ אין Skill Scanner לפני התקנה
  • ❌ אין Block Mode אוטומטי
📋 Best Practices לפי Anthropic

⚙️ Hooks — המנגנון הרשמי לאבטחה
Anthropic מספקים מנגנון Hooks ב-code.claude.com/docs/en/hooks — זהו הבסיס הרשמי של EHZ-SEC-AI. הם מספקים אפילו דוגמת קוד רשמית ב-GitHub.
סוג Hook מתי רץ יכולות שימוש אבטחתי
PreToolUse לפני ביצוע Tool allow deny ask defer חסימת פקודות מסוכנות לפני ביצוע
PostToolUse אחרי ביצוע Tool לוגים, Feedback ל-Claude ניטור, סריקת קבצים שנכתבו
PostToolUseFailure כשTool נכשל לוגים, התראות זיהוי ניסיונות כושלים
HTTP Hook כל אירוע שליחה ל-endpoint חיצוני ← זה בדיוק EHZ-SEC-AI!
קוד רשמי מ-Anthropic — HTTP Hook לשרת חיצוני (בדיוק הארכיטקטורה של EHZ-SEC-AI):
// settings.json — Hook עם HTTP endpoint
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "http",
        "url": "http://localhost:3010/hooks/pre-tool-use",  ← EHZ Collector
        "timeout": 30,
        "headers": { "Authorization": "Bearer $CCSM_TOKEN" }
      }]
    }]
  }
}
🐙 Reference Implementation רשמי: Anthropic פרסמו bash_command_validator_example.py — Hook שחוסם grep ו-find ומאלץ שימוש ב-ripgrep. מדגים בדיוק את Pattern שבו EHZ-SEC-AI משתמש.
📊 OpenTelemetry — ניטור רשמי ל-Enterprise
Anthropic מספקים תמיכה מלאה ב-OpenTelemetry דרך CLAUDE_CODE_ENABLE_TELEMETRY=1. מתאים לארגונים עם Prometheus / Grafana / Splunk / Datadog.
claude_code.session.count
מספר Sessions שנפתחו
claude_code.token.usage
שימוש ב-Tokens (input/output/cache)
claude_code.cost.usage
עלות ב-USD לכל session
claude_code.lines_of_code.count
שורות קוד שנוספו/הוסרו
claude_code.tool_result
אירוע לכל tool שהושלם (success/fail + duration)
claude_code.tool_decision
כל החלטת accept/reject של tool
OTEL_LOG_TOOL_DETAILS=1
הפעלה: פרמטרים מלאים של כל tool (פקודות Bash, נתיבי קבצים)
OTEL_LOG_USER_PROMPTS=1
תיעוד תוכן הפרומפטים (כבוי כברירת מחדל)
⚠️ הבדל מ-EHZ-SEC-AI: OpenTelemetry = מוכוון ROI ועלויות, לא אבטחה. אין זיהוי אנומליות, אין Hardening Levels, אין Secrets Scanner. EHZ-SEC-AI משלים את הפער הזה.
חלק ב׳ — OWASP Top 10 סיכוני LLM (2024)
OWASP פרסמו רשימת 10 הסיכונים הקריטיים ליישומי LLM. לכל סיכון — כיצד EHZ-SEC-AI מטפל בו.
LLM01
Prompt Injection
תוקף מחדיר הוראות זדוניות דרך קלט לגרום לLLM לבצע פעולות לא מורשות.
🛡 EHZ-SEC-AI: זיהוי context חשוד ב-PreToolUse Hook
LLM02
Insecure Output Handling
אי-ולידציה של פלט LLM מאפשרת RCE, XSS, SQL Injection במערכות downstream.
🛡 EHZ-SEC-AI: PostToolUse סריקת קוד שנכתב
LLM03
Training Data Poisoning
זיהום נתוני אימון גורם לתגובות שפוגעות באבטחה, דיוק, התנהגות אתית.

LLM04
Model Denial of Service
הצפת המערכת בפעולות עתירות משאבים גורמת לשיבוש ועלויות.

LLM05
Supply Chain Vulnerabilities
תלות ברכיבים/שירותים/datasets פגומים פוגעת בשלמות המערכת.
🛡 EHZ-SEC-AI: Skill Scanner — Hash + YARA לפני כל התקנה
LLM06
Sensitive Information Disclosure
חשיפת מידע סודי בפלטים יוצרת סיכונים משפטיים ותחרותיים.
🛡 EHZ-SEC-AI: Secrets Scanner — סיסמאות, API keys, כרטיסי אשראי, ת"ז
LLM07
Insecure Plugin Design
Plugins שמטפלים בקלט לא מהימן ללא הגבלות גישה מאפשרים RCE.
🛡 EHZ-SEC-AI: MCP Hook monitoring + Skill pre-install scan
LLM08
Excessive Agency
אוטונומיה בלתי מוגבלת מובילה לתוצאות לא מכוונות שפוגעות בפרטיות ואמינות.
🛡 EHZ-SEC-AI: Hardening Levels 0-3 + Block Mode (Milestone 10)
LLM09
Overreliance
אמון עיוור בפלטי LLM יכול להוביל לפגיעויות אבטחה והחלטות שגויות.

Model Theft

EHZ-SEC-AI ממלא פער אמיתי
אין מוצר בשוק שמנטר Claude Code ספציפית דרך Hooks. Anthropic מספקים את ה-מנגנון (Hooks API) אבל לא מוצר מלא. EHZ-SEC-AI הוא השכבה שחסרה.

Anthropic מאשרים את הגישה שלנו
הארכיטקטורה של EHZ-SEC-AI — HTTP Hook ל-Collector מקומי — היא בדיוק מה שAnthropic מדגימים בתיעוד הרשמי שלהם. אנחנו בכיוון הנכון.
⚡
OpenTelemetry כאלטרנטיבה משלימה
OTel של Anthropic מצוין לניטור ארגוני (עלויות, tokens, ROI) אבל לא לאבטחה. שווה לשקול לשלב OTel + EHZ-SEC-AI — כל אחד עושה משהו אחר.

7/10 סיכוני OWASP מכוסים
EHZ-SEC-AI מטפל ב-70% מסיכוני OWASP LLM Top 10. הנותרים (Data Poisoning, Model Theft) הם ברמת Anthropic ומחוץ לשליטת המשתמש.

יתרון על פני Lasso ו-Spectral
המוצרים הפעידים (Lasso, Spectral) הם Enterprise בתשלום. EHZ-SEC-AI חינמי, מותאם אישית ל-Claude Code, ועם System Tray + Telegram שאין במוצרים אחרים.

סיכון WebDAV ב-Windows
Anthropic מזהירים במפורש: אל תאפשר WebDAV ב-Windows — Claude יכול לבצע network requests לhosts חיצוניים ולעקוף את Permission System. EHZ-SEC-AI צריך לכסות את הסיכון הזה.
המלצות — כיצד לשפר EHZ-SEC-AI בהתבסס על המחקר
📋 5 שיפורים מיידיים שהמחקר חשף
🔴 חשוב — הוסף לRules
חסימת WebDAV paths ב-Windows (\\* prefix). Anthropic מזהירים במפורש מסיכון זה.
🟠 שקול שילוב OTel
הוסף תמיכה ב-OpenTelemetry output — כך EHZ-SEC-AI יכול להזין נתונים גם ל-Grafana/Prometheus ארגוניים.
🟡 Prompt Injection Detector
OWASP LLM01. הוסף זיהוי patterns של Prompt Injection בתוכן שClaude מקבל (WebFetch results, file reads).
🔵 MCP Hook Monitoring
Anthropic מציינים MCP כסיכון אבטחה. הוסף matcher ל-mcp__*__write* לניטור כל כתיבה דרך MCP servers.
🟢 ConfigChange Hook
Anthropic מספקים ConfigChange hook — התראה כשמישהו משנה את settings.json. חשוב לאבטחה ב-Domain.
